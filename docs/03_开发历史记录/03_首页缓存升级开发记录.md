# 03_首页缓存升级开发记录

## 需求背景

首页 Redis 整页缓存（commit `b9b43a6`）虽然把 HTML 骨架写入了缓存，但链接数据仍要前端发起 2 次 API 请求（`/api/links`、`/api/categories`），其中 `/api/categories` 实测 0.44s，偏慢。需要把"首屏可见时间"再压一档。

## 关联提交

- `b9b43a6` perf(cache): 添加首页 Redis 整页缓存
- `0114c99` perf(cache): 首页整页缓存内嵌公开链接数据，减少 67% HTTP 请求
- `efe1dde` fix(cache): 补充 batchMove 后清除首页缓存

## 实现要点

- `HomeController::renderAndOutput()` 预查询分类和 `is_private = 0` 的链接，连同 `privacyEnabled` 标志一起传给视图
- `resources/views/home.php` 在 `<script>` 中嵌入 `window.__NAV_DATA__`；Alpine 组件 `init()` 优先用内嵌数据渲染，新增 `syncLinks()` 在 `privacyEnabled=true` 时后台异步补齐隐私链接
- 隐私安全：缓存中**永不**内嵌隐私链接；`privacyEnabled` 只标识系统是否启用隐私空间，不含用户 session 状态
- 缓存失效：4 个 Category 写方法 + 6 个 Link 写方法（store/update/destroy/batchMove/batchDestroy/batchUpdate/refreshIcon）末尾追加 `PageCacheHelper::forget('home')`
- 缓存键：`nav:page:home`（Redis DB3），TTL 6 小时

## 测试或验证收口

- 匿名浏览器窗口访问首页：未启用隐私时不再出现 `/api/categories`、`/api/links` 请求
- 已验证用户刷新页面：内嵌公开链接立即渲染，0.1s 后后台 API 补齐隐私链接
- 写操作后首页缓存自动失效，新链接/分类/排序立即可见
- 登录用户绕过缓存走原 API 流程，未受内嵌数据影响
- 降级兼容：Redis 不可用时自动回退到直渲染

## 本阶段收益

- 首页 HTTP 请求数：3 → 1（隐私未启用）/ 2（隐私已启用，后台异步）
- 首屏可见时间：从等 JS+API（~0.5s）压缩到内嵌数据立即渲染（<0.1s）
- `/api/categories` 在未启用隐私的访问路径中调用频次降为 0
- 缓存写方法覆盖完整，10 个 Controller 写路径均触发失效
