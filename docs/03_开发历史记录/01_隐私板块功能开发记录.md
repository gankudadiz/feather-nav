# 01_隐私板块功能开发记录

## 需求背景

个人导航首页需要给一些链接加上"隐私"标记，默认不公开显示；首页提供密码入口，验证后展示隐私链接，Session 内免重复输入。

## 关联提交

- `e5cdedf` feat: 添加隐私板块功能
- `e8c84cb` chore: 补充隐私空间密码环境变量到 .env.example

## 实现要点

- 数据库：`links` 表新增 `is_private TINYINT(1)`，建立 `idx_is_private` 索引
- 配置：`.env` 增加 `PRIVATE_SPACE_PASSWORD`（留空则功能不启用）
- 后端：新增 `POST /api/verify-private` 写 `$_SESSION['private_verified']`；`GET /api/links` 在未验证时仅返回 `is_private = 0` 的链接
- 前端：首页搜索栏旁增加"🔒 隐私空间"按钮 + 密码弹窗；`resources/views/admin.php` 链接表单和列表中加入隐私字段

## 测试或验证收口

- 未输入密码时不显示隐私链接（已上线实测）
- 正确密码后隐私链接正常显示，刷新页面保持验证
- Admin 面板可标记链接为隐私
- `.env` 未配置密码时所有链接公开

## 本阶段收益

- 个人导航具备轻量级隐私隔离，无需重写权限系统
- 密码校验走 Session，密码不会持续在网络传输
- 与后续首页缓存方案衔接：缓存中永不内嵌隐私数据，隐私链接由客户端后台 `GET /api/links` 补齐
